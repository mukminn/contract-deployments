# Deploy ber4 Token

Status: PENDING

## Description

This task deploys the ber4 ERC20 token as a pure L2 native token (no L1 counterpart).

- **Token Name**: ber4
- **Token Symbol**: ber4
- **Token Type**: Standard ERC20 (L2 native)

## Procedure

### 1. Setup

1. Create a `.env` file in this directory with the following variables:
   - `OPTIMISM_COMMIT`: Commit hash of Optimism contracts
   - `BASE_CONTRACTS_COMMIT`: Commit hash of Base contracts
   - `L2_RPC_URL`: L2 RPC URL (Base mainnet)
   - `PRIVATE_KEY`: Private key of the deployer account
   - `ETHERSCAN_API_KEY`: Etherscan API key for contract verification

2. Install dependencies:
   ```bash
   make deps
   ```
   Or manually:
   ```bash
   forge install OpenZeppelin/openzeppelin-contracts
   ```

### 2. Simulate Deployment

Before deploying, simulate the transaction:

```bash
make simulate
```

Or directly:
```bash
forge script script/DeployBer4Token.s.sol:DeployBer4Token --rpc-url $L2_RPC_URL -vvvv
```

### 3. Deploy

Deploy the token:

```bash
make deploy
```

Or directly:
```bash
forge script script/DeployBer4Token.s.sol:DeployBer4Token \
  --rpc-url $L2_RPC_URL \
  --broadcast \
  --verify \
  --etherscan-api-key $ETHERSCAN_API_KEY \
  -vvvv
```

### 4. Verify

The deployment script includes verification. After deployment, verify the contract on Etherscan if needed.

## Notes

- This is a pure L2 token (no L1 counterpart)
- The token uses OpenZeppelin's ERC20 implementation
- **Initial Supply**: 1,000,000 tokens (1M * 10^18) will be minted to the deployer address upon deployment
- Ensure the deployer account has sufficient funds for gas fees on Base mainnet
- The token will be deployed directly without using a factory
- Update [FEES.md](./FEES.md) after deployment with contract address and transaction details
